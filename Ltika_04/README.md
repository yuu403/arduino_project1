# 🚦 歩行者ボタン付き信号機制御システム

## 📌 概要
Arduinoを用いて、車両用信号（赤・黄・緑）を制御しつつ、  
歩行者ボタンによって信号の状態を変化させるシステムを実装した。

単純な時間制御に加え、入力（ボタン）と状態遷移を組み合わせた  
実践的な組み込みプログラムとなっている。

---

## ⚙️ 動作

通常時は以下の順で信号が切り替わる：

- 🔴 赤（5秒）
- 🟢 緑（5秒）
- 🟡 黄（2秒）
- → 繰り返し

### 🚶‍♂️ 歩行者ボタン
- 緑信号中にボタンを押すと、次の状態を強制的に黄へ移行
- その後、通常通り赤へ遷移

---

## 🛠 使用技術

- デジタル入出力制御
- `millis()`による時間管理（ノンブロッキング）
- 状態遷移（ステートマシン）
- エッジ検出（ボタン入力制御）

---

## 🧠 設計のポイント

### ① 状態遷移の導入
各信号（赤・緑・黄）を状態として管理し、  
`switch`文で制御することで、拡張しやすい構造とした。

---

### ② millisによる非同期処理
`delay()`を使用せず、`millis()`で時間管理を行うことで、  
ボタン入力など他の処理と並行可能な設計にした。

---

### ③ ボタンのエッジ検出
ボタンが押されている間に複数回反応することを防ぐため、  
「押した瞬間のみ検出」するロジックを実装した。

---

### ④ 現実に近い信号制御
ボタン押下時に即座に赤へ切り替えるのではなく、  
「緑 → 黄 → 赤」という自然な遷移を維持するよう設計した。
